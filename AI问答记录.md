## 三、调试与修改过程

### 问题1：数据下载失败

**现象**：使用`torchvision.datasets.FER2013`下载时报错，提示文件不存在。

**排查过程**：检查了官方文档，发现`FER2013`不支持自动下载，需要预先放好CSV文件。又尝试了Kaggle API下载，但国内访问慢。

**解决方案**：从GitHub镜像手动下载`fer2013.csv`，用自定义`Dataset`类加载。


### 问题2：图片显示中文为方框

**现象**：matplotlib绘图时，中文标题显示为方框。

**排查过程**：查了matplotlib的字体配置，发现默认字体不支持中文。

**解决方案**：
```python
plt.rcParams['font.sans-serif'] = ['Microsoft YaHei', 'SimHei']
plt.rcParams['axes.unicode_minus'] = False
```


### 问题3：训练时loss不下降

**现象**：模型训练了10轮，loss几乎没变，一直在1.5左右。

**排查过程**：检查了数据预处理、模型定义、学习率。发现是数据增强太强（随机擦除），导致模型学不到有效特征。

**解决方案**：降低数据增强强度，去掉ColorJitter，只保留翻转和旋转。


### 问题4：Gradio清除按钮报错

**现象**：点击清除按钮抛出`ValueError: A function didn't return enough output values`。

**排查过程**：检查了`clear_btn.click()`的绑定，发现`outputs`有2个组件但函数只返回1个值。

**解决方案**：
```python
def clear_all():
    return None, "等待上传图片..."
```


### 问题5：模型加载报KeyError

**现象**：`app.py`里加载模型时报`KeyError: 'unexpected key "conv1.weight"'`。

**排查过程**：打印了`state_dict().keys()`，发现模型定义和保存时不一致——`app.py`里的类名和训练时不同。

**解决方案**：确保`app.py`中模型类的定义与训练时完全一致。


### 问题6：测试集准确率低于预期

**现象**：训练集准确率78%，测试集只有60%。

**排查过程**：分析了混淆矩阵，发现"恐惧"和"惊讶"经常混淆，"厌恶"样本太少。检查数据增强发现可能太强导致训练不足。

**解决方案**：增加训练轮数到30轮，添加权重衰减，用更稳定的学习率调度。

### 改进效果总结

1. **准确率提升**：+5.33%（60.32% → 65.65%）
2. **开心类表现最好**：85% → 88%（容易识别）
3. **厌恶类表现最差**：42% → 48%（样本少，特征不明显）
4. **恐惧和惊讶容易混淆**：类间相似度高
5. **数据增强有效**：提高了泛化能力
6. **GAP有效**：减少过拟合，参数量控制合理
